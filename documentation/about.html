About

More About EMG Tools and Its Development

Built with simplicity in mind.

EMG Tools started as a solution to seeing new fellow graduate students around me building their own scripts to process their EMG data with. At the time, not many of us could code very well and there was always uncertainty with what we had programmed. There was a need for a standard where we could have confidence in the result instead of worrying about having made a programming error. Wanting to solve this problem and at the same time improve my own programming skills, I set out to create a solution.

In over my head.

Included in the group of those who couldn't code very well was me. Throughout the process of learning to create a GUI application in Matlab I had a lot of help from another graduate student. Since the beginning, I've always acknowledged the mentorship of Dianne Ikeda in both helping with building EMG Tools' predecessor and with helping me in my early days of learning to program. Thank you!

Continued development.

EMG Tools has always been a playground for me to exercise my programming abilities. While I hope that someone somewhere can find it useful, its primary purpose has always been a fun project for me to learn coding with. It started in the first year of my master's degree and I continued to release revisions throughout my PhD. Since beginning my programming journey in Matlab, I've expanded into Python, JavaScript, HTML, CSS, SQL, as well as a handful of frameworks and libraries. As I improve, I return to EMG Tools to try and make enhancements.


Thanks for taking a look!

- Christian Balkovec
